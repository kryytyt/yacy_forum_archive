Das Blog-HTML ist absichtlich so gestaltet, dass es ...

Date: 2013-11-27 15:34:05

Das Blog-HTML ist absichtlich so gestaltet, dass es keine absolute Positionierung hat, und keine Längen- oder Breitenangaben, damit das auf wirklich jedem noch so ranzigen Schrottbrowser anständig angezeigt werden kann. Der Browser macht die Schrift so groß, dass man was erkennen kann, und bricht dann halt die Zeilen früher um. So würde man sich das jedenfalls denken. In der Praxis tun das die getesteten Browser auch — bis auf Safari. Apple hat es mal wieder mit Anlauf, Kopf nach vorne, Volldampf gegen die Wand verkackt. Daher schickt das Blog jetzt ein proprietäres Apple-Tag mit, wenn jemand mit Safari vorbeikommt.

So und jetzt will ich kein Geheule mehr hören.
