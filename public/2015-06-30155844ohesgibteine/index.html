Oh, es gibt eine neue TLS-Implementation in C, s2n, ...

Date: 2015-06-30 15:58:44

Oh, es gibt eine neue TLS-Implementation in C, s2n, von Amazon. 6000 Zeilen Code, allerdings ist das ohne die Cipher-Bausteine, die nehmen sie dann von libcrypto von openssl (oder anderen Optionen). Das API sieht schön aus, die Unit-Tests sehen schön aus. Client Mode ist noch nicht implementiert, sie haben noch kein Certificate Checking implementiert. Das ist also rein für SSL-Server im Moment. Gut, kleine Abstriche gibt es ja immer. Das zweite Argument zu s2n_config_add_dhparams sollte const char* sein, finde ich, nicht char*. Aber hey.

Mich stört im Moment, dass der non-blocking Mode dir zwar sagt, er ist noch nicht fertig, aber er sagt dir nicht, ob er jetzt schreiben oder lesen wollte und das geblockt hätte. Das ist für ein Event-basiertes Framework wie in gatling aber notwendig. Mal schauen, ob da nur die Doku scheiße ist, oder auch der Code :)
